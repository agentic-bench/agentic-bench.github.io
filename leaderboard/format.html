Submission Format

Unified llm-comments JSONL format required for all benchmark submissions.

Overview

Each submission is a JSONL file (one JSON object per line), where each line represents one diff (pull request) that was reviewed. The filename determines the agent identity: 

{agent_id}_{YYYYMMDD-HHMM}.jsonl

Examples:
  my-agent_20260301-0900.jsonl
  rovodev-cli-0-12-16_20251105-0800.jsonl

Place the file in benchmarks/{benchmark_name}/llm-comments/.

Top-level Fields (per line / per diff)

{
  "diff_id":    "airflow_issue_10616_pr_10617_l_6bfba8c5",  // required — matches dataset.jsonl
  "submission": { ... },                                      // required — agent metadata
  "trajectory": { ... },                                      // required — cost/usage data
  "has_reviews": true,                                        // required — whether agent produced any comments
  "reviews":    [ ... ]                                       // required — list of review comments (may be [])
}
FieldTypeDescription
diff_id required string Must match a diff_id in the benchmark's dataset.jsonl.
submission required object Agent identity — see Submission block below.
trajectory required object Token usage and cost for this diff — see Trajectory block below.
has_reviews required boolean true if the agent produced at least one review comment for this diff.
reviews required array List of review comment objects. May be [] if agent decided no comment is needed.

Submission Block

"submission": {
  "agent_id":    "my-agent",                    // required — stable identifier (used for leaderboard key)
  "model":       "claude-3-5-sonnet-20241022",  // MANDATORY — base model name (used for cost calculation)
  "timestamp":   "20260301-0900",               // required — YYYYMMDD-HHMM (must match filename)
  "extra": {                                    // optional — any additional metadata
    "config_name": "default",
    "version":     "1.0.0",
    "description": "My agentic reviewer using chain-of-thought"
  }
}
FieldTypeDescription
agent_id required string Stable agent name. Latest submission for this agent_id replaces previous ones in the leaderboard.
model required string MANDATORY — Name of the underlying LLM (e.g. gpt-4o, claude-3-5-sonnet-20241022). Must be recognized by genai_price for cost calculation. Submission will be rejected if this field is missing.
timestamp required string Must match the timestamp portion of the filename. Format: YYYYMMDD-HHMM.
extra optional object Any additional metadata. Stored as-is; some evaluators may inspect it.

Trajectory Block

"trajectory": {
  "input_tokens":  12450,   // required — total input tokens consumed for this diff
  "output_tokens": 843,     // required — total output tokens produced for this diff
  "total_tokens":  13293,   // optional — total tokens (input + output)
  "steps":         7,       // required — number of agent steps / LLM calls
  "extra": {                // optional — any additional usage fields
    "tool_calls":  5,
    "requests":    3
  }
}

Note: Cost metrics (input_costs, output_costs, total_costs) are automatically calculated by the evaluation pipeline based on token counts and the model name specified in the submission block. You should not include cost values in your submission.

Note: The extra field can contain additional trajectory data like tool_calls and requests which may be used by evaluators for deeper analysis.

FieldTypeNotes
input_tokens required numberSummed across all LLM calls for this diff.
output_tokens required numberSummed across all LLM calls for this diff.
total_tokens optional numberTotal tokens (usually input_tokens + output_tokens).
steps required numberNumber of agentic steps / tool calls / LLM invocations.
extra optional objectAny additional usage data (cache tokens, latency, etc.).

Reviews Array

Each element in reviews is a single code review comment:

"reviews": [
  {
    "file":    "airflow/models/dagbag.py",  // required — relative file path
    "line":    321,                          // required — integer line number (1-based)
    "comment": "This approach of removing modules from sys.modules..."
                                             // required — the review comment text
  }
]
FieldTypeNotes
file required string Relative path within the repository (e.g. src/foo/bar.py). No leading slash.
line required integer Line number in the file. Must be a single integer — no line ranges. A ±5 window is used for fuzzy location matching.
comment required string The review comment text produced by the agent.
confidence optional number Confidence score (0.0-1.0) for this review comment. Optional metadata for analysis.

Full Example (one line / one diff)

{
  "diff_id": "airflow_issue_10616_pr_10617_l_6bfba8c5",
  "submission": {
    "agent_id":  "my-agent",
    "model":     "claude-3-5-sonnet-20241022",
    "timestamp": "20260301-0900",
    "extra": { "version": "1.0.0" }
  },
  "trajectory": {
    "input_tokens":  12450,
    "output_tokens": 843,
    "total_tokens":  13293,
    "steps":         7
  },
  "has_reviews": true,
  "reviews": [
    {
      "file": "airflow/models/dagbag.py",
      "line": 321,
      "comment": "Removing modules from sys.modules here can cause unintended side effects if other parts of the application hold references to these modules. Consider using a more targeted approach."
    },
    {
      "file": "tests/models/test_dagbag.py",
      "line": 257,
      "comment": "The line `del sys.path[:1]` mutates the global sys.path, which may affect other tests running in the same process. Consider using a context manager or restoring the original path in a teardown."
    }
  ]
}

Filename Convention

{agent_id}_{YYYYMMDD-HHMM}.jsonl

Rules:
- agent_id    : lowercase, hyphens allowed, no underscores (underscores separate agent_id from timestamp)
- YYYYMMDD    : 8-digit date (year, month, day)
- HHMM        : 4-digit time (hour, minute) in 24h format
- .jsonl      : one JSON object per line, UTF-8 encoded

Re-submitting with the same agent_id but a newer timestamp will replace the previous entry in the leaderboard (latest-wins deduplication).

Validation Tips

  • Every line must be valid JSON. Use python3 -c "import json; [json.loads(l) for l in open('file.jsonl')]" to check.
  • All diff_id values must appear in the benchmark's dataset.jsonl.
  • Diffs that your agent did not review may be omitted from the file — the evaluator will count missing diffs as not accomplished.
  • Diffs where your agent decided no comment is needed should still be included with "reviews": [] and "has_reviews": false.
  • comment_line must be a positive integer. Do not use line ranges.
  • All mandatory trajectory fields must be present and numeric (use 0 if unknown, not null).
